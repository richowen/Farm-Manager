# v0.5.0 Plan — Calendar & To-do Unification

## Background

Discussion during v0.4.x development surfaced a clearer mental model for the
app's organising features:

| Concept | Description | Current state |
|---|---|---|
| **Past events** | Activity logs attached to a field/location (fertiliser, grazing, worming, etc.) | ✅ `/events` — works well |
| **Calendar / planned activities** | Future-dated items that belong on a calendar (vet visit, cattle market, planned worming) — should sync to Google/Apple Calendar | ⚠️ Exists as "Tasks" but poorly labelled; iCal feed already works |
| **To-do list** | Things to do when you get around to it — no specific date, just a checklist | ⚠️ Partially covered by Pins (to-do status) but not surfaced cleanly |

---

## Goals

1. **Rename Tasks → Calendar** everywhere in the UI (nav, page title, TopBar icon,
   MobileNav, iCal token in Settings)
2. **Redesign the Calendar page** to feel like a planner, not a task manager
3. **Surface to-do pins** as the undated to-do list — unify with the Calendar
   page or give them a dedicated lightweight view
4. **Extend iCal feed** to optionally include future-dated field events
   (e.g. auto-exporting a worming event logged with a future `occurred_at`)

---

## Scope

### 1. Rename Tasks → Calendar

Files to update:
- `src/routes/tasks/+page.svelte` → rename route to `src/routes/calendar/+page.svelte`
  (or keep the URL as `/tasks` to avoid breaking iCal token URLs, just change labels)
- `src/lib/components/MobileNav.svelte` — tab label "Tasks" → "Calendar", update icon
- `src/lib/components/TopBar.svelte` — link title/aria-label
- `src/routes/settings/+page.svelte` — iCal section heading
- `src/routes/tasks/+page.svelte` — `<title>`, `<h1>`
- API routes stay as `/api/tasks` (no breaking change)

### 2. Calendar page redesign

Current tabs: `Overdue | Due today | Upcoming | Done`
New tabs: `Upcoming | Past | To-do` (or just `Planned | Done`)

- **Planned** tab: all items with `due_at > now()`, sorted ascending (soonest first)
- **Done** tab: completed items
- **To-do** sub-section (or separate tab): to-do pins loaded from `GET /api/pins?status=todo`
  - Completing a pin here calls `PATCH /api/pins/:id` with `{ status: 'done' }`
  - Shows pin title/category and "View on map" link
  - No due date — just pending indefinitely

Remove snooze buttons (`+1 day`, `+1 week`) — they only make sense for strict
deadlines, which the user doesn't want.

Keep recurrence for worming-schedule tasks (just don't expose it in the manual
create form — it's set programmatically).

### 3. Calendar create form

- Remove required due date → make date **optional** (collapsible "Set a date?" toggle)
- If no date is set, store `due_at = now()` and don't display the date in the UI
- Remove recurrence from the manual create form (still set programmatically)
- Keep: title, optional date, optional location, optional notes

### 4. iCal feed extension (optional / stretch)

Currently the iCal feed (`/calendar.ics`) exports tasks by `due_at`.
Consider also exporting field events where `occurred_at > now()` as
`VEVENT` entries with the location name in the summary.

This would let you log a future "Vet visit — North Field" event on the map and
have it appear in Google Calendar automatically.
